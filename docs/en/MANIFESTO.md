# MANIFESTO

## Mission

Reality API Demo is the public face of the Reality API research effort.
Its mission is to present the project's core hypothesis, evidence trail, and research outputs in a transparent and accessible way.

## Core Statement

We study the hypothesis that physical reality behaves like a computational system.
This demo does not claim final truth; it documents observations, hypotheses, and analysis in a falsifiable and inspectable format.

## Why the Demo Exists

- Make the project understandable without private repository access
- Share current research state in a read-only public format
- Provide an auditable snapshot of observations and references
- Build trust through openness and consistency

## Principles

1. Evidence first: claims must connect to observations and references.
2. Falsifiability: hypotheses should be testable and challengeable.
3. Clarity over hype: speculative ideas are labeled as speculative.
4. Bilingual access: all user-facing content is available in EN and TR.
5. Reproducibility: static data snapshots allow independent review.

## Scope of This Repository

- Public, static, read-only showcase
- No data entry, no admin workflows, no private operations
- No database runtime dependency

## Relationship to the Full Platform

The full platform lives in the private repository and includes database-backed CRUD, admin tools, and operational workflows.
This demo is intentionally constrained to communication, transparency, and discoverability.
